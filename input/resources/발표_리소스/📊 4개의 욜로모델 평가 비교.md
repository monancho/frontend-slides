### 🏆 어떤 상황에서 어떤 모델이 좋을까요? (분석 포인트)

#### 1. "작은 쓰레기까지 꼼꼼하고 정확하게 잡아야 해요!" ➔ **네번째 모델 (8m, 61ep)**

- **이유:** 체급이 큰 Medium 모델이라 쓰레기의 복잡한 패턴을 깊게 이해합니다. 특히 네번째 모델은 로컬에서 기초를 다진 후 코랩에서 조심스럽게 재학습(Warm Restart)되었기 때문에, 겹쳐있거나 찌그러진 쓰레기(Debris)의 경계선을 아주 타이트하게(mAP50-95 지표 우수) 잘 잡아냅니다.
- **Confusion Matrix 분석:** 쓰레기를 먼지나 배경으로 오해하는 비율이 가장 적게 나타납니다.

#### 2. "CCTV나 라즈베리파이 같은 기기에서 실시간으로 렉 없이 돌아가야 해요!" ➔ **첫번째(8n) 또는 두번째(8s) 모델**

- **이유:** 네번째 모델(8m)은 똑똑하지만 꽤 무겁습니다. 고성능 GPU가 없는 일반 컴퓨터나 소형 엣지(Edge) 기기에서 실시간 영상을 분석해야 한다면 Nano(8n)나 Small(8s) 모델을 써야 화면이 끊기지 않습니다.
- **추천:** 실시간 속도가 최우선이면 **1번(8n)**, 약간의 속도 저하를 감수하고 정확도를 챙기려면 2번(8s)을 선택하세요.

#### 3. "학습 안정성과 신뢰도 (Loss 그래프 분석)" ➔ **네번째 모델 (8m, 61ep) 승리**

- **3번 모델(91ep)의 패착:** 코랩에서 백지상태로 너무 강하게 학습을 밀어붙인 결과, 초중반에 오차율(Loss)이 하늘로 솟구치는 대혼돈을 겪었습니다. PR Curve에서도 찌그러진 구간이 보일 만큼 모델이 불안정합니다.
- **4번 모델(61ep)의 성공:** 훈련 손실(Train Loss)과 검증 손실(Val Loss) 그래프가 바닥을 향해 아주 예쁘고 부드럽게 미끄러집니다. 모델이 억지로 정답을 외우지 않고(과적합 방지), 진짜 '쓰레기의 본질'을 이해하며 학습했다는 강력한 증거입니다.
